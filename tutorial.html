Hello hello and welcome all! Today I will be going over the basics of using terminal controls as used on this site. This guide is designed for guests that have seldom/never used a terminal interface before.

The whole website turned into a love letter of sorts (to the field && my degree) so there are references I added that may not make sense to my audience without that background. However, I have done my best to make the main content accessible through this guide, and provided (hopefully!!) enough to find out more!

If, after reading this guide you are still confused, or have some feedback about what could be done better - please reach out! With that out of the way, lets begin!

We'll be looking at the following commands. You may notice that some of them have values in brackets '[]' after - what's all that about?
cd [dir| ]
help [name| ]
list [dir| ]
look [value]
quit
run [name]
show[name]
tree[dir| ]

Please type in 'more' to proceed to the next section...

So, for each command in the list you will see its name, followed by its arguments. You can think of the arguments as though they are variables in a math equation:
2x + 6y cannot be solved if you don't know what x and y are*! So, the values in brackets represent what can be plugged in for that x and y.

Luckily for us, there is only ever 1 potential argument 'x' if you will! Thus, anything you pass in will become 'x'. Except, just 'x' isn't very descriptive, so instead we'll see words reminding us what is allowed as an input!

dir - Short for directory. We'll go more in depth in just a bit ;), for now, all you need to know is that is describes a location!
name - Going back to the algebra example, this is what you would call 'x'. It could describe a command, like 'list' or 'help', but it can also describe a file, like 'art.txt'
value - This is like name++, or Name: The Expanded Edition. Pretty much anything you can use as a name you can also use as a value, but you can also describe things that aren't explictly named. For example, you could describe the website (like, 'background' or 'home'). Alternatively, you could choose to describe its author ('Natalie', 'dsa fanboy', etc.)
blank, or no brackets - For commands without brackets, or with the '| ]' at the end, this is a sign telling us that that it doesn't take a value, or that there is a special default value
